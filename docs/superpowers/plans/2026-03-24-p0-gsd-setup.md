# P0 — GSD Setup Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Install GSD and populate all 4 project files with FacilityPro context so every future session starts with zero re-explanation.

**Architecture:** GSD is installed globally via npx, generating a `.gsd/` directory in the project root. The 4 markdown files inside it are the persistent memory layer — PROJECT.md (identity), REQUIREMENTS.md (full PRD), ROADMAP.md (P0–P7 phases), STATE.md (current progress). Once populated, each Claude Code session loads these files before doing any work.

**Tech Stack:** Node.js (npx), GSD CLI (`get-shit-done-cc`), Markdown, Git

```markdown
# FacilityPro Mobile — Project

## What This Is
FacilityPro Mobile is the mobile client for an enterprise Facility Management & Services (FMS) platform. The web app and Supabase backend are already built (~80% schema complete). This app is a companion mobile interface — it does not build a new backend; it connects to the existing Supabase project.

## Tech Stack
- Framework: Expo SDK 55 + React Native 0.83
- Navigation: React Navigation 7 (native-stack + bottom-tabs)
- Backend: Supabase (direct — same project as web app)
- Auth: Supabase Auth + expo-local-authentication (OTP + biometric)
- State: Zustand (auth, role, session)
- Server state: TanStack Query (data fetching, caching, mutations)
- Offline: AsyncStorage + expo-secure-store (sensitive data)
- Location: expo-location + expo-task-manager (geo-fence, background patrol)
- Camera: expo-camera + expo-image-manipulator + expo-image-picker
- Notifications: expo-notifications + FCM + Supabase Edge Functions (SMS fallback)

## User Roles (7 total)
Each role gets its own isolated navigator after login:
1. Security Guard → GuardNavigator
2. Security Supervisor → SupervisorNavigator
3. Society Manager → ManagerNavigator
4. HRMS Staff (payroll-administered) → HRMSNavigator
5. Service Staff (AC Tech / Pest Control / Delivery Boy) → ServiceNavigator
6. Buyer → BuyerNavigator
7. Supplier/Vendor → SupplierNavigator

## Key Constraints
- Visitor approval is SMS-only — no Resident app
- Android 10+ / iOS 14+ minimum
- Guard SOS system must be fully offline-capable (queue + retry)
- Background location requires two-step Android 10+ permission flow
- Session expires after 8 hours of inactivity

## Source of Truth
- Full design spec: `docs/superpowers/specs/2026-03-24-gsd-facilitypro-mobile-design.md`
- Full PRD: see REQUIREMENTS.md in this directory
```

```markdown
# FacilityPro Mobile — Requirements

## Authentication & Onboarding (P1)
- Mobile Number + OTP login (primary method)
- Role-based routing after login — each role directed to its own home screen
- Biometric unlock (Face ID / Fingerprint) for subsequent sessions
- Session expiry: auto-logout after 8 hours of inactivity
- First-time setup: mandatory profile photo capture (guards)
- Device registration + push notification permission request
- Geo-fence calibration: app walks user to primary work location to set baseline GPS

## Security Guard App (P2)

### Home Screen
- SOS Panic Button: full-width RED button, single tap, no confirmation step
- Clock In / Clock Out: selfie-based, camera opens automatically, photo timestamped + GPS-stamped, geo-fence validated
- Daily Checklist shortcut with badge showing pending count
- Emergency Contacts: one-tap quick-dial list

### SOS Panic System
- Single tap triggers alert — no confirmation (delay is dangerous)
- Payload: Guard ID, Name, GPS (lat/lng), Timestamp, Auto-captured photo
- Recipients: Society Manager (push + SMS) + Security Supervisor (push)
- Alert stays active until manager acknowledges from dashboard
- Continuous GPS streaming begins on SOS trigger until resolved
- OFFLINE: queue alert locally, fire the moment network available

### Daily Operational Checklist
- Items configured in admin portal, pushed to guard at shift start
- Each item: Yes/No toggle or numeric value (e.g. water tank level %)
- Photo evidence upload per item (optional by default, mandatory for critical items)
- Parking Lights: log time ON (evening) and OFF (morning)
- Water Supply: motor pump status + tank level reading
- Gate/Shutter Check: confirmation secondary gates locked
- Auto-reminder push if checklist not started by 9:00 AM
- Completed checklist submitted and locked — edits require supervisor override
- OFFLINE: writable offline, syncs on reconnect

### Inactivity / Patrol Alert
- If guard GPS does not change >10m in 30 minutes → Inactivity Alert to Supervisor
- Guard receives vibration + sound nudge 5 minutes before threshold
- Guard can tap "I am on duty" to reset timer (for legitimately stationary posts)

### Visitor Entry Logging
- Captures: Name, Phone Number, Photo (camera), Vehicle Number (optional)
- Selects destination flat from Society Family Database (searchable dropdown)
- System sends SMS to resident: "Dear Resident, [Name] is at the gate for Flat [X]"
- Visitor approval is SMS-only — no resident app
- Daily/Frequent Visitors: separate list for maids, drivers, milkmen — select from list

### Emergency Contact Directory
- One-tap call: Police, Fire Brigade, Ambulance, Electrician, Plumber
- Configured by Society Manager — guards cannot edit
- OFFLINE: always accessible without network

## Security Supervisor & Society Manager App (P3)

### Live Monitoring Dashboard
- Real-time guard location map, all on-duty guards, refreshed every 60 seconds
- Checklist status board: green (complete) / red (pending) per guard
- Panic log feed: active SOS alerts, tap to view guard location + acknowledge
- Visitor stats: total entries today / this week, drill-down by gate
- Staff attendance: check-in/check-out log with timestamps + GPS

### Ticket Management
#### Employee Behaviour Ticket
- Select staff member from dropdown
- Category: Sleeping on Duty / Rudeness / Absence from Post / Grooming+Uniform / Unauthorized Entry
- Severity: Low (Warning) / Medium (Serious) / High (Critical)
- Incident description text + optional photo/video evidence
- Date & time auto-captured

#### Material Ticket (Quality & Quantity Check)
- Triggered when delivery vehicle logged at gate by guard
- Manager receives push notification to inspect goods
- Material Ticket form: Check Quantity (ordered vs received) + Check Quality (Good/Damaged/Expired/Leaking)
- Condition status, photo evidence, batch number
- If Approved: items added to inventory
- If Rejected: Return Ticket auto-generated

## HRMS — Mobile Features (P4)

### Smart Attendance with Geo-Fencing
- Selfie attendance: tap Check In → front camera → photo → GPS validated → check-in recorded
- Geo-fence: check-in only permitted within 50m of registered Company Location
- Auto punch-out: if GPS moves outside geo-fence >15 consecutive minutes → flag record + notify supervisor
- Check-out: same selfie + GPS process

### Leave Management
- Submit: Leave Type (Sick/Casual/Paid), start date, end date, reason
- Supervisor receives push to Approve or Reject
- Employee gets in-app notification with decision
- Real-time leave balance shown on staff home screen

### Payslip & Payroll View
- View and download monthly payslip as PDF
- Earnings breakdown: Basic, HRA, Special Allowance, Overtime
- Deductions breakdown: PF, PT, ESIC
- Historical payslips for last 12 months

### Document Vault
- View uploaded ID documents: Aadhar, PAN, Voter ID
- Security staff: view PSARA certificate, Police Verification status
- Upload replacement/updated documents from camera or gallery

## Service Workflow Apps (P5)

### AC Technician App
- Receive service request notification
- Start Work: GPS + timestamp logged automatically
- Upload Before photo
- Log parts used from Equipment Supply (item + quantity → deducted from stock)
- Upload After photo
- Complete: work order closed, Society Manager notified

### Pest Control Technician App
- PPE Checklist: must check off Mask, Gloves, Eye Protection, Apron — cannot proceed until complete
- Attendance with selfie + GPS geo-fence (same as HRMS)
- Chemical Request: submit material request (chemical + quantity); manager approves in-app
- Service Proof: Before and After photos of treated areas
- Resident Notification: system auto-sends SMS/push to affected flats

### Delivery Boy App
- View active delivery orders assigned to this role
- Update status: Picked Up → In Transit → Delivered
- Capture delivery proof photo at destination
- Buyer + Admin receive push on each status change

## Buyer App (P6)
- Place Order: service category, grade/role/product, quantity, duration, submit
- Order Tracking: live status (Requested → Accepted → Indent Sent → PO Raised → Dispatched → Delivered)
- Accept / Reject proposed quotation or order terms
- Invoice & Payment: view and download Sale Bill; mark payment settled
- Feedback: star rating + free-text; required to close order
- Push alerts for every status change

## Supplier / Vendor App (P6)
- Indent Inbox: view incoming indents with full line-item details
- Accept / Reject indent (reject requires reason)
- PO Acknowledgement: confirm receipt of Purchase Order
- Dispatch Update: mark goods dispatched (vehicle number + estimated arrival)
- Received Note Upload: delivery note or challan as photo or PDF
- Bill Submission: generate + submit Supplier Bill; track payment status (Pending → Paid)
- Push alerts for new indents, PO issuance, payment updates

## Notification Architecture (P7)

All 11 routes — see full table in design spec Section 10.

| # | Notification | Priority | DND Bypass | SMS Fallback |
|---|---|---|---|---|
| 1 | SOS / Panic Alert | CRITICAL | Yes | Yes |
| 2 | Visitor at Gate | High | No | Yes (SMS only, no push) |
| 3 | Inactivity Alert | High | No | Yes |
| 4 | Checklist Reminder | Medium | No | No |
| 5 | Order Status Change | Medium | No | No |
| 6 | New Indent | High | No | Yes |
| 7 | Material Delivery | High | No | Yes |
| 8 | Leave Decision | Medium | No | No |
| 9 | Payslip Ready | Low | No | No |
| 10 | Pest Control Alert | High | No | Yes (SMS only) |
| 11 | Low Stock Alert | Medium | No | No |

## Non-Functional Requirements
- Android 10+ / iOS 14+
- App launch to home screen: <3 seconds on mid-range Android
- Selfie attendance photo capture and upload: <5 seconds on 4G
- Push notification delivery: <10s standard; <5s for SOS
- All API calls HTTPS with certificate pinning
- Role-based access: RLS policies per Supabase table
- Visitor personal data purged after 90 days
- Minimum tap target: 44×44dp
- High-contrast mode support
- Text size follows OS accessibility settings
- Offline data at-rest encryption for sensitive fields (SOS queue, emergency contacts)
- 8-hour inactivity auto-logout

## Out of Scope (Mobile)
- Resident app — visitor approval is SMS-only
- Master data configuration (web portal only)
- Financial reporting
- Purchase Order generation
- Payroll calculation
- Advertising and printing management
- Society Family Database administration
- E2E automated tests (post-feature-complete)
```
